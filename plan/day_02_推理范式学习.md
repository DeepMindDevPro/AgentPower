Day2 聚焦的四种推理范式，恰好是赋予 Day1 中“规划”与“反思”两大心智模块以具体思维策略的方法论。它们解答了“思维应该如何展开”这一核心问题，且各自对应着《人工智能：一种现代方法》里不同的决策框架。

---

## 一、范式精析

### 1. ReAct (Reasoning & Acting)
**定义**：将推理（Reasoning）与行动（Acting）交错编织。智能体**想一步、做一步、观察结果、再想下一步**，形成“思考-行动-观测”的循环。
- **核心特点**：
  - 外部信息驱动：行动调用工具（搜索、查询）获取新事实，再注入推理。
  - 事实更新：每一次行动都可能修正、扩充内部知识，避免幻觉。
  - 可解释性强：输出的不是孤立的答案，而是带引证的思维轨迹。
- **最适场景**：需要**与外部环境频繁交互、信息不完全**的任务。如事实核查、多跳问答、网页导航、动态场景决策。
- **AIMA 对应**：**基于模型的反射型智能体** + **序贯决策**。智能体在部分可观测环境中，交替进行“状态更新（感知）”与“规划/行动”，维持一个内部世界模型。

### 2. CoT (Chain-of-Thought)
**定义**：在给出最终答案前，显式生成一系列**中间推理步骤**。这些步骤是纯语言逻辑推导，不调用外部工具，是一条**线性、连贯的思维链**。
- **核心特点**：
  - 线性分解：将复杂问题逐步拆分为简单子问题。
  - 内部推理：仅靠模型自身知识完成，不依赖外部反馈。
  - 无分支探索：单路径推理，如果某一步错误，整个链条崩塌。
- **最适场景**：**纯粹的多步逻辑推理、数学计算、常识推理**，且模型内部已包含所需的全部知识。如算术、符号推理、因果推断。
- **AIMA 对应**：**基于目标的智能体**，通过**搜索/规划**找到一条从当前状态到目标的行动序列。CoT 就是大脑内部的一次“模拟计划”，不依赖执行器反馈。

### 3. ToT (Tree of Thoughts)
**定义**：将推理过程建模为**在思维空间中的树搜索**。每一步可生成多个候选“思维节点”，再通过**评估（打分/投票）** 决定扩展哪个分支，最终找到最优的思维路径。
- **核心特点**：
  - 广度探索：每个中间步骤考虑多种可能性。
  - 前瞻与回溯：可向前看几步，评估后剪枝或回溯。
  - 搜索策略：BFS/DFS/蒙特卡洛等，可动态分配计算资源。
- **最适场景**：**需要创造性、策略规划、多方案比较**且**判别标准明确**的任务。如创意写作、策略游戏规划、需要系统性探索的数学证明。
- **AIMA 对应**：**基于效用的智能体**，使用**与或图搜索、博弈树搜索**。思维树本质上是在内部状态空间中进行的启发式搜索，目标是找到最大化效用（评分）的推理链。

### 4. Self-Refine (自我迭代优化)
**定义**：智能体首先生成一个**初始输出**，然后对自己产出的内容进行**自我反馈（批评/建议）**，再根据反馈**迭代改进**，循环往复，直至满足质量要求。
- **核心特点**：
  - 迭代打磨：不是推翻重来，而是在上一版本基础上精修。
  - 内外反馈结合：反馈完全由模型自己生成（也可辅以外界信号），但评估标准内嵌于智能体的“品味”中。
  - 广泛适用：可叠接到 CoT、ToT 之上，作为输出的后处理打磨环。
- **最适场景**：**生成与创作**类任务，尤其当质量可被语言化评估时。如文章润色、代码优化、翻译改进、论证加强。
- **AIMA 对应**：**反思模块的实例化**。这正是不断根据性能度量（自我生成的评价）修正行为的过程，是对“基于效用的反思型智能体”的一种具体实现。

## 二、核心差异与适用场景速查

| 范式 | 思维结构 | 与环境交互 | 计算资源 | 典型适用场景 | 失败模式 |
|------|---------|------------|----------|------------|---------|
| **ReAct** | 推理-行动循环图 | 强（调用工具） | 按需调用 | 事实查证、多跳QA、交互式决策 | 陷入无效循环、行动冗余 |
| **CoT** | 线性链 | 无 | 单路径生成 | 数学推理、逻辑谜题、代码逻辑 | 一步错步步错 |
| **ToT** | 树状分支 | 无（或内部评分） | 多条路径并行探索 | 创意写作、策略规划、方案设计 | 搜索空间爆炸、评估不准 |
| **Self-Refine** | 迭代螺旋 | 无（或有外部评价） | 多次迭代 | 文本润色、代码优化、翻译 | 过拟合到自我偏好、循环无提升 |

**选择口诀**：
- 缺外部信息活不了 → **ReAct**
- 纯粹烧脑一步步推 → **CoT**
- 需要多选一、全局最优 → **ToT**
- 已有草稿想磨成精品 → **Self-Refine**

### 组合使用
现代高级智能体往往不是单用一种。例如：
- **ReAct + CoT**：在 ReAct 的每个“推理”步骤中使用 CoT 进行深度分析。
- **ToT + Self-Refine**：在 ToT 的每个叶节点，用 Self-Refine 打磨候选回答，再择优。
- **ReAct + Self-Refine**：对 ReAct 生成的整体轨迹进行自我批评，然后重新规划。

## 三、回归基石：与 AIMA 第2章架构的映射

将这两种推理范式放入 Day1 的五组件架构中，它们的用武之地一目了然：

- **CoT / ToT** 主要运行在 **规划** 模块内。它们是将高层目标转化为可执行步骤的“思维算法”。
- **Self-Refine** 主要运行在 **反思** 模块内。它提供了一种具体的元认知过程：生成自评 → 提出修改 → 重新执行。
- **ReAct** 则是一种**整体调度策略**，它将规划、执行、记忆更新、反思编织成一个紧密的实时闭环。它的“推理”部分可以调用 CoT/ToT，它的“行动”部分调用工具，它的“观测”部分直接写入记忆。
- **记忆** 在这四种范式中都是背景支持：CoT/ToT 的上下文是记忆；ReAct 的每一步都读写记忆；Self-Refine 的迭代版本存储在记忆中。

可以说，Day1 给出了骨架，Day2 的这四种范式就是让骨架**动起来的神经回路**。掌握它们的选择与组合，你就能为不同任务装配出具备相应“思维风格”的智能体。
